# Coffee AI Boss 2D → 3D 办公室重构计划（基于真实代码勘察）

> 版本 v2。基于对 Claw3D-main 3D 内核与本仓库逐文件阅读后重写，纠正了 v1 关键错误（见「v1 错误纠正」）。

## 一、勘察结论（事实，非推测）

### A. 本项目当前状态
- 2D 可视化只有 **一个文件** `app/static/index.html`（约 17.7KB），由 `app/main.py` 的 `index()` 以 FileResponse 返回，挂在 `/`；静态资源挂在 `/static`。
- `app/main.py` 路由：`/chat`、`/agents/register`、`/skill/register`、`/agents`、`/agents/{id}/heartbeat`、`/agents/{id}/actions`、`/visualization/events`、`/admin/restaurant-state`、`/ws/visualization`、`/user/{id}`、`/history/{id}`、`/orders/{id}`、`/status`、`/`。
- 事件契约（visualization_service.py）：`/ws/visualization` 连接即发 `scene.snapshot`，之后广播单事件 `{event_id,type,agent_id,payload,correlation_id,created_at}`。角色集合 `{customer,waiter,cashier,barista,manager}`；动作集合 `{enter_scene,walk_to_counter,walk_to_table,take_order,prepare_coffee,deliver_order,show_message,leave_scene,error}`。
- **存在半成品 `frontend/`**：是 **Phaser 2D + Colyseus** 脚手架（phaser、colyseus.js、BootScene），构建输出到 `app/static/game/`，**不是 3D**。另有 `colyseus-server/`。这些属于旧 2D 方案，删除阶段一并清理。
- 事件来源：`/chat`(web_dialog) 与 A2A Skill 路径都调用 `_publish_visualization_event` 广播；`AgentProfile.sprite_seed`/`status` 驱动角色外观。
- `domain_constants.py` 管控订单/支付/身份状态，不可引入 ad hoc 字符串。

### B. 关键现实：项目当前【没有】用户登录/注册机制
- `User` 表只有 `user_id/nickname/balance/taste_preference/时间戳`，**没有 password_hash、token、session/cookie/JWT**。
- 全仓搜不到 login/register/密码/cookie/auth 的用户态实现；`/chat` 直接用 `ChatRequest.user_id` 匿名走单（user_id 由前端传入）。
- `/agents/register`、`/skill/register` 只为 **A2A 消费方发 API token**，不是用户账号。
- 结论：你说的「原有的登录注册机制」在本仓库并不存在。要在 3D 场景跑登录/注册，必须**新建一套账号体系**（见决策点 1）。

### C. Claw3D 3D 内核（逐文件读懂）
可直接移植的「纯渲染内核」（依赖极少）：
- `core/constants.ts`：SCALE=0.018，画布→Three 世界坐标映射常量、行走速度、agent 尺度。
- `core/geometry.ts`：`toWorld(cx,cy)→[x,y,z]`、家具占地、导航阻塞元数据、包围盒。
- `core/navigation.ts`：`buildNavGrid()` + `astar()` —— 自包含 25px 网格 A* 寻路（含对角穿角修正），渲染/移动解耦的核心。
- `objects/agents.tsx`：`AgentModel` —— box 几何体拼的低多边形人偶（身体/四肢/头/五官/头发），`useFrame` 每帧从 `agentsRef` 读 `{x,y,facing,state,frame,status}` 做动画（走路摆臂、坐姿、眨眼、说话气泡、错误红圈脉冲）。
- `scene/environment.tsx`：`FloorAndWalls`(地板/墙)、`WallPictures`(画框/旗杆)。
- `systems/sceneRuntime.tsx`：`GameLoop({tick})`(每帧只调一次 tick)、`FloorRaycaster`(指针→地面投影)、`SpotlightEffect`。

不能直接搬、必须**按本项目事件契约重写**：
- 7249 行 `RetroOffice3D.tsx` 是「胶水+业务外壳」，深度耦合 Claw3D 的 gateway/store/onboarding/远程办公室/技能市场/Hermes，**严禁整体复制**。只取其中「移动模拟 tick()」逻辑，改成由本项目可视化事件驱动。
- 依赖项本地化桩：
  - `createDefaultAgentAvatarProfile`(lib/avatars/profile) → 本地写按 agent_id/sprite_seed 确定性生成肤色/发色/服装的 stub。
  - `objects/types.ts` 的 `AgentModelProps` → 本地复制。
  - `core/district.ts` 区域常量 → 本地定义单办公室布局，去掉远程办公室/城市分区。

### D. v1 错误纠正
1. v1 说「frontend 新建」——实际已存在且是 **Phaser 2D**，必须先清理再建 3D。
2. v1 说「后端契约一律不改」——但你新增了登录需求，而项目无账号体系，故**必须新增账号后端**（不破坏现有下单/支付/事件契约）。
3. v1 说「保留 legacy-2d.html 回退」——与「彻底删除 2D」冲突；v2 按 git 历史归档，不留运行态。

## 二、目标产物
1. 新 3D 前端模块（React+Vite+TS+three+@react-three/fiber+@react-three/drei），默认入口即 3D 办公室，每个 agent 是入职员工（工位/走动/工作/气泡/异常），用于实时监控追踪。
2. 一套账号登录/注册（后端 + 3D 前端），登录后在 3D 场景内继续下单/可视化。
3. 一个大屏展示(dashboard)：聚合 `admin/restaurant-state` 的今日单量/金额、员工状态、最近事件流，与 3D 场景同屏。
4. 彻底移除旧 2D 方案（`app/static/index.html`、`frontend/` Phaser 脚手架、`colyseus-server`）。

## 四、领域映射（咖啡店 → 办公室入职员工）
角色 → 工位类型（坐标在 roleMap.ts，画布坐标系经 toWorld 投影）：
- `barista`→咖啡工位（操作咖啡机）  `cashier`→前台/收银工位  `waiter`→巡场/助理（多走动）  `manager`→主管工位（独立角落）  `customer`→访客/新入职待分配（门口接待区）

动作 → 3D 行为：
- `enter_scene`→从门走入，astar 寻路到工位
- `walk_to_counter`/`walk_to_table`→沿 navGrid 寻路到目标点
- `take_order`/`prepare_coffee`→工位执行「工作」动画（status=working 绿圈脉冲+键盘/机器）
- `deliver_order`→工位间移动并交接
- `show_message`→头顶气泡显示 payload.text（复用 AgentModel speechBubble）
- `leave_scene`→寻路回门口离场
- `error`→status=error 红圈脉冲+眉毛下垂（AgentModel 已内置 isError 分支）

未知 role/action 一律前向兼容（忽略，不报错）。

## 五、决策点（执行前需确认）

**决策点 1 — 登录/注册方案（项目当前无账号体系）**
推荐：新增 `UserAccount` 表（username/email + password_hash 用 passlib[bcrypt]，会话用签名 httpOnly cookie），新增 `/auth/register` `/auth/login` `/auth/logout` `/auth/me`，幂等迁移脚本 `scripts/migrate_user_accounts.py`。3D 前端加 `/login` `/register`，登录后 cookie 带入 `/chat`。现有 `/chat` 仍可匿名以保兼容。
> 需你拍板：同意新建账号体系（推荐），还是改用「nickname 取 user_id」的轻量入口？

**决策点 2 — 彻底删除 2D 的时机**
推荐：3D 验收通过后一次性删除 `app/static/index.html`、`frontend/`(旧)、`colyseus-server/`，`index()` 改返回 3D 入口；删除前 `git mv` 归档到 `_archive/2d-legacy/` 便于回溯。

## 六、分阶段任务拆分（建议执行顺序）
- 阶段 0｜清理与脚手架：备份归档旧 2D；清空 frontend 旧 Phaser 与 colyseus-server 引用，重建 React+Vite+TS；装 three/r3f/drei/react-router/zod。
- 阶段 1｜移植 3D 渲染内核（纯前端）：复制 core/{constants,geometry,navigation,types}；复制 objects/agents.tsx+types.ts，替换 avatar/profile 依赖为本地 stub；精简 environment.tsx；复制 systems/{sceneRuntime,cameraLighting}.tsx。静态验收：build 出 3D 办公室非空白。
- 阶段 2｜接线后端事件：visualizationSocket 连 ws；agentStore 维护状态机；tick 移植简化；roleMap 实现映射。验收：真实下单一次，3D 出现 enter→walk→工作→deliver→leave。
- 阶段 3｜账号登录/注册（依赖决策点 1）：后端 app/auth/ + UserAccount + 幂等迁移 + /auth/*；前端 /login /register + AuthProvider。验收：注册→登录→3D 下单成功，刷新保持登录。
- 阶段 4｜大屏展示：Dashboard.tsx 聚合 admin/restaurant-state + 3D 缩略 + 员工状态墙；路由 /dashboard。
- 阶段 5｜彻底删除 2D 与收尾：删除旧 2D 文件（已 git 归档）；index() 改 3D 入口；全量回归。

## 七、约束与禁区
- MySQL 唯一关系库，Redis 唯一内存后端：禁止 SQLite/fakeredis/内存回退。
- 连接信息只从 `.env` 的 MYSQL_*/REDIS_* 读，禁止硬编码密码/token/连接串到代码/文档/页面/日志。
- 账号表改动走幂等迁移，不破坏现有 order/User/agent_profile 等表。
- 下单/支付/订单表逻辑（order_service / skill_order_service / models.order）不动；事件契约（角色/动作集合、事件结构、ws 路由）不破坏，前端只读。
- 脏仓库：开工前列计划改动文件并 `git status --short -- <文件>`，不 reset/revert 任何无关改动。
- 严禁搬 Claw3D 的 gateway/Hermes/demo-gateway/Skills Marketplace/onboarding。
- 7249 行 RetroOffice3D.tsx 严禁整体复制，只移植渲染内核 + 简化 tick。
