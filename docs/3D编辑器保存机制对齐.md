## 背景（为什么需要这个改动）

用户反馈：3D 咖啡厅场景（`/3d/scene`）的家具编辑器"看不到保存按钮，编辑后不生效"。

对照移植源头 Claw3D（`D:\temp\EVOMAP\Claw3D-main`）后发现根因：**当前项目用"手动分散保存"，遗漏了键盘微调操作的保存，导致编辑后刷新即丢失。** 这与 Claw3D 的"单一 debounced autosave"模式不对齐。

## 根因分析（两库对照）

### Claw3D `RetroOffice3D.tsx`（7248 行，全文件仅 1 处 `saveFurniture`）

```js
// RetroOffice3D.tsx:3147-3154 —— 监听 furniture 变化，300ms debounce 后自动落盘
useEffect(() => {
  const timeoutId = window.setTimeout(() => {
    saveFurniture(furniture, storageNamespace);
  }, 300);
  return () => {
    window.clearTimeout(timeoutId);
  };
}, [furniture, storageNamespace]);
```

所有操作（`handleFloorClick` / `handleFloorMove` / `handleDelete`(:4904) / `handleReset`(:4916) / 键盘处理）**只 `setFurniture`**，由上面这个 effect 统一保存。

> **Claw3D 也没有"保存"按钮** —— 编辑工具栏只有 Pencil/Check（切换编辑）、Reset、Delete、Show Objects。它完全依赖 autosave effect。

### 当前项目 `frontend/src/screens/OfficeScene.tsx`（6 处分散 `saveFurniture`）

| 行号 | 操作 | 是否保存 | 说明 |
|------|------|---------|------|
| 233 | 放置墙体完成 | ✓ `saveFurniture(next)` | 正常 |
| 254 | 放置家具完成 | ✓ `saveFurniture(next)` | 正常 |
| 264 | 拖动移动完成 | ⚠️ `saveFurniture(furniture)` | 用闭包旧值，stale 隐患 |
| 292 | 方向键移动 | ✗ 只 `setFurniture` | **遗漏，刷新丢失** |
| 293 | PgUp/Dn 抬升 | ✗ 只 `setFurniture` | **遗漏，刷新丢失** |
| 302 | `[` `]` 旋转 | ✗ 只 `setFurniture` | **遗漏，刷新丢失** |
| 305 | Delete 键删除 | ✓ | 正常 |
| 454 | 删除按钮 | ✓ | 正常 |
| 464 | 恢复默认布局 | ✓ | 正常 |

**结论**：用键盘微调家具（移动 / 旋转 / 抬升）后，改动只留在 React 内存里，`localStorage` 不更新 → 刷新页面丢失 → 表现为"编辑后不生效"。**这 3 处遗漏（292 / 293 / 302）是核心 bug。**

## 方案：移植 Claw3D 的 debounced autosave

把"手动分散保存"改为"单一 debounced autosave effect"，与 Claw3D 完全一致。这样**所有来源的 furniture 变更都自动持久化**，彻底消除遗漏，并顺手修掉 line 264 的闭包 stale 隐患。

### 修改文件：`frontend/src/screens/OfficeScene.tsx`（唯一改动点）

**1. 新增 debounced autosave effect**（紧跟现有 `useEffect(() => { sim.setFurniture(furniture); }, [sim, furniture])` 之后，约 line 96 后）：

```ts
// Debounced autosave (ported from Claw3D RetroOffice3D). ANY furniture change
// — drag, keyboard move/rotate/elevate, delete, reset — flushes to localStorage
// 300ms after the last edit, so per-action saveFurniture calls aren't needed
// and keyboard micro-edits no longer vanish on reload.
useEffect(() => {
  const id = window.setTimeout(() => saveFurniture(furniture), 300);
  return () => window.clearTimeout(id);
}, [furniture]);
```

**2. 移除 6 处手动 `saveFurniture` 调用**（保留外层 `setFurniture` 不动）：

- line 233（placing wall 分支内的 `saveFurniture(next)`）
- line 254（placing item 分支内的 `saveFurniture(next)`）
- line 264（moving 分支的 `saveFurniture(furniture)`）
- line 305（Delete 键的 `saveFurniture(next)`）
- line 454（删除按钮的 `saveFurniture(next)`）
- line 464（恢复默认的 `saveFurniture(DEFAULT_FURNITURE)`）

effect 会在这 6 处的 `setFurniture` 触发后自动接管保存。`saveFurniture` 的 import 保留（effect 用到）。

## 不做的事（避免过度设计）

- **不**移植 Claw3D 的后端快照同步（`PUT /api/office/layout`，`RetroOffice3D.tsx:3156-3182`）—— 那是 Claw3D 多房间远程协作特性，本项目是单咖啡厅 localStorage，无需求。
- **不**移植 `storageNamespace` —— 本项目只有 `default` 单命名空间（`persistence.ts` 默认参数已处理）。
- **不**新增"保存"按钮 —— Claw3D 本身也没有，靠 autosave。

## 可选增强：保存状态指示器

Claw3D 无此反馈，但用户明确表达了"不知道有没有保存"的困惑。可在 autosave effect 旁加一个轻量状态指示：

```ts
const [saveState, setSaveState] = useState<"idle" | "pending" | "saved">("idle");
// effect 内：setSaveState("pending") → setTimeout 里 saveFurniture + setSaveState("saved")
```

在左上角信息条（`OfficeScene.tsx:404-407` 那个"Coffee AI Boss · 3D 咖啡厅"面板）追加一行 `布局：已保存 ✓` 或 `保存中…`。**此项可选**，若只想要功能对齐可跳过。
